## 2024-06-12 - [Optimize Dockerfile layers and image size]
**Learning:** Found that multiple `apt-get` commands in separate `RUN` directives without `--no-install-recommends` and `rm -rf /var/lib/apt/lists/*` caused unnecessary bloat in the Docker image layers. The lack of `--no-cache-dir` on `pip install` further contributed to a larger image size and potentially cached unnecessary files. Docker layer caching can also be broken due to uncombined `apt-get update` and `apt-get install`.
**Action:** Next time, always consolidate `apt-get update` with `apt-get install` commands into a single `RUN` layer to maximize caching efficiency. Always append `--no-install-recommends` for debian based images to avoid installing optional dependencies not explicitly requested. Conclude all `apt-get install` operations with `rm -rf /var/lib/apt/lists/*` in the same RUN layer to remove fetched package lists directly, saving noticeable space. Finally, always append `--no-cache-dir` to `pip install` directives to prevent accumulating tarball caches locally during CI builds unless explicitly caching in a multi-stage Dockerfile environment (though the pip cache won't be in the final stage here, it's a good generic standard).
